# 근본 원인 우선 진단

표면 증상에 맞춰 코드를 고치면 회귀가 생긴다. 이 스킬은 "고치기 전에 진짜 원인을 잡는다"를 강제한다.

## 절대 규칙

1. **이 워크플로가 끝나기 전에는 어떤 Edit/Write도 하지 않는다.** 디버그 로그 삽입은 진단 도구이므로 허용 (단계 3에서만).
2. **첫 가설이 그럴듯하다고 바로 받아들이지 않는다.** 항상 3개 이상의 후보를 나열하고 증거로 검증한다.
3. **사용자가 "이게 원인일거야"라고 추측을 던지더라도** 그대로 받지 않고 검증 과정을 거친다.

## 워크플로

### 1단계: 증상 정리

사용자에게 받은 증상을 한 문장으로 정리하고, 다음을 명시한다:
- 무엇이 일어나야 하는가 (expected)
- 무엇이 일어나고 있는가 (actual)
- 언제부터 / 어느 환경에서 발생하는가

이 단계에서 부족하면 진행하지 말고 사용자에게 물어본다.

### 2단계: 코드 경로 끝까지 읽기

증상이 일어나는 코드 경로를 **엔드투엔드로** 추적한다. 클라이언트 → 서버 → 외부 API → 응답 → 상태 반영까지. 중간 어느 단계가 잘못됐는지 모르는 채로 가설을 세우지 않는다.

이 단계의 산출물: "데이터/제어 흐름 N단계" 목록.

### 3단계: 후보 가설 3개 랭킹

증거와 함께 가설 3개를 제시한다. 각 가설마다:

```
가설 A (확률 ★★★): <한 줄 요약>
  - 근거: <코드 라인·로그·관측값으로 뒷받침>
  - 반증 방법: <어떤 관측이 있으면 이 가설이 틀린가>
```

- 가장 그럴듯한 것을 1번에 두되, 가능성이 낮아 보여도 2~3번도 진지하게 적는다
- 가설끼리는 서로 다른 레이어에 걸치도록 (예: 클라이언트 상태 vs 서버 응답 vs 외부 의존성)

### 4단계: 검증 — 디버그 로그가 필요하면 여기서만

가장 그럴듯한 가설을 데이터로 확인한다:
- 기존 로그/캐시/저장소 상태를 먼저 본다
- 부족하면 **임시 디버그 로그를 추가하고 사용자에게 출력을 부탁한다**
- 로그 추가는 검증 도구로만 사용한다. 이때 "추가한 로그는 진단 끝나면 제거"한다고 명시한다

검증 결과가 가설과 어긋나면 → 2번 가설로 이동 (1번 가설을 방어하려 들지 않는다).

### 5단계: 진짜 원인 확정 후 ExitPlanMode

원인이 확정되면 plan 모드로 들어가 수정안을 제시한다. plan에는 반드시 포함:
- 확정된 근본 원인 (어떤 증거로 확정했는지)
- 수정할 파일과 정확한 변경 내용
- 이 수정이 영향을 줄 수 있는 다른 케이스 (회귀 위험)

사용자 승인 전에는 Edit/Write 하지 않는다.

### 6단계: 디버그 로그 정리

수정이 끝나면 단계 4에서 넣은 임시 로그를 모두 제거한다. 별도 요청이 없어도 자동으로.

## 안티패턴 — 하지 말 것

- "이게 원인 같으니 고쳐볼게요" 하고 바로 Edit (← 가설 검증 없음)
- 첫 가설이 틀렸을 때 그 가설을 변형해서 또 시도 (← 가설 방어, 2번 후보로 이동해야 함)
- 사용자가 본 클라이언트 로그만 보고 서버측을 확인 안 함 (← 한쪽만 본 상태로 결론)
- 코드 잔존물·옛 데이터·"과거 코드 흔적"을 원인으로 지목 (← 정말 그게 원인이라는 증거가 따로 있어야 함)
- 회귀 위험 검토 없이 "수정 완료" (← 단계 5의 영향 케이스 누락)

## 사용 예시

사용자: "주식 동기화가 안 돼"
→ 단계 1: 증상 정리 ("동기화 트리거 시 가격이 안 바뀜")
→ 단계 2: syncTodayStockPrices → /api/finance → KIS API 응답 → setStock → 클라이언트 응답 적용 경로 추적
→ 단계 3: 가설 A(KIS 토큰 만료), 가설 B(슬롯 키 미스매치), 가설 C(클라이언트 outdated 판정 누락)
→ 단계 4: 서버 콘솔 로그·캐시 파일 확인 → 토큰 만료 확정
→ 단계 5: plan으로 수정안 제시
→ 단계 6: 임시 로그 제거

## 메타 규칙

이 스킬을 실행하는 동안 사용자가 "그냥 빨리 고쳐"라고 재촉해도, **단계 1~3은 건너뛰지 않는다.** 단계 4 검증은 가설 강도에 따라 축약 가능. 단, 검증 없이 단계 5로 점프하지 않는다.
