# 전체 기능 QA 실행

`.claude/_knowledge/qa-full-test-plan.md`를 **단일 출처(source of truth)** 로 삼아 전체 기능 회귀 테스트를 수행한다.
원칙: **라이브 프리뷰 검증 제외**(CLAUDE.md). 자동·코드레벨 검증은 직접 수행하고, 앱 수동 항목(👤)은 사용자 실행 체크리스트로 산출한다.

## 워크플로

### 0. 계획 로드
- `.claude/_knowledge/qa-full-test-plan.md`를 읽어 Phase 0~3 구성을 확인한다. (계획이 갱신되면 자동 반영)
- 사용자가 특정 기능(F-XRAY 등)이나 범위를 지정했으면 그 섹션만, 없으면 전체를 대상으로 한다.

### 1. Phase 0 — 자동 베이스라인
순서대로 실행하고 결과를 표로 보고:
```bash
npx tsc --noEmit
npx vitest run
npm run lint
npm run build
```
- 하나라도 실패하면 출력을 첨부하고, 관련 기능 Phase로 내려가 코드 경로를 추적한다(추측 패치 금지).

### 2. Phase 1 — 기능별 코드레벨 점검
- 대상 기능들의 ⚙ 항목을 코드 경로 추적으로 검증(데이터 흐름 끝까지). 계산·캐시 키·직렬화·가드 로직 위주.
- 엣지·경계 케이스를 표로 정리하고, 위반·의심 지점을 식별.

### 3. Phase 2 — 회귀 위험 레지스트리(R1~R8)
- 변경 범위와 무관하게 R1~R8을 빠르게 스캔(특히 변경된 파일이 닿는 항목).

### 4. Phase 3 — 산출물 보고
- 발견 이슈를 **P0(데이터 손실/크래시) / P1(기능 오작동) / P2(UX·표시)** 로 분류. 회귀면 R# 매핑.
- 각 이슈에 근본 원인 + 재현 경로(파일:라인) 명시.
- 👤 수동 항목은 사용자가 앱에서 실행할 체크리스트로 별도 정리.
- **이 스킬은 진단·보고까지만 수행한다. 코드 수정은 사용자 승인 후 별도로.**

## 규칙
- 발견 이슈를 추측으로 단정하지 않는다 — 코드·테스트·로그로 근거를 댄다.
- 계획 문서에 없는 새 검증이 필요하면 수행하되, 재사용 가치가 있으면 `qa-full-test-plan.md`에 반영을 제안한다.
- 응답은 한국어·간결. 결과(통과/실패·이슈)만 명확히.
