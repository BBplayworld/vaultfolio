# Plus 구독 모델을 위한 용어(Asset) 정의 및 구조

향후 Plus 유료 구독 모델을 도입하기 위해, 구독 및 과금이 귀속될 공통 단위인 **Asset**과 관련 기술 용어를 정의하고 표준화합니다.

## 1. 표준 용어 및 코드 식별자 매핑

| 용어                         | 정의                                                                     | 코드 식별자             | 비고                                   |
| ---------------------------- | ------------------------------------------------------------------------ | ----------------------- | -------------------------------------- |
| **Asset (전체 자산 / 금고)** | 사용자의 **전체 자산 데이터 묶음**. 동기화·공유·구독의 공통 단위         | `AssetData` (평문)      | **1 user = 1 asset**, UI 노출명 "금고" |
| **assetId**                  | Asset 고유 식별자. 서버 저장 키·동기화 주소·서명 대상                    | `assetId`               | 값 변경 없이 그대로 유지               |
| **AssetEnvelope**            | 서버(Redis/파일)에 저장되는 E2EE 암호화 봉투                             | `AssetEnvelope`         | 기존 `VaultEnvelope` 대체              |
| **asset-item (자산 항목)**   | asset 내부 개별 보유 (주식/부동산/코인/현금/대출)                        | item `.id`              | 현행 유지                              |
| **Cloud Sync (동기화)**      | assetId로 asset을 E2EE 저장/복원. **새 기기 공유 + 지속(양방향) 동기화** | `cloud-sync/*`          | **Plus 대상**(지속 동기화가 차별점)    |
| **Share (공유)**             | 자산 스냅샷 단기 토큰. **새 기기 공유(1회 핸드오프)만 가능**             | `share key`, `owner_id` | asset 흐름과 별개                      |

---

## 2. 데이터 흐름도

```
구독·과금 단위 = Asset (assetId)
   └─ Cloud Sync : assetId → csync:asset:{assetId} (Redis, E2EE, 무기한)   ← Plus 대상
   └─ 동기화 링크 : #asset=<assetId>&theme=...
   └─ 인증        : 금고암호 → PBKDF2(salt) → Ed25519 서명 (서버는 pubKey + 암호문만 보관)
Share (공유)    = 콘텐츠 해시 기반 단기 토큰 (asset과 독립, owner_id 추적)  ← 구독 무관
asset-item      = asset 내부 배열 요소(.id) — 구독 단위 아님
```

---

## 3. 기능 포지셔닝

| 능력                                           | Share (공유) |  Cloud Sync (동기화)  |
| ---------------------------------------------- | :----------: | :-------------------: |
| 새 기기로 자산 옮기기(1회 핸드오프)            |      ✅      |          ✅           |
| 지속 양방향 동기화(자동 백업·복원·다기기 일치) |      ❌      | ✅ (Plus 요금제 가치) |

두 기능 모두 "새 기기 공유" 기능을 제공하지만, **지속 동기화는 Cloud Sync(Plus)에서만** 지원한다는 점을 메뉴 및 안내 문구에서 명확히 구분합니다.
